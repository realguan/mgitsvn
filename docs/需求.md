### 需求背景
项目采用微服务架构，涉及多个服务模块，代码由 SVN 托管。开发团队更习惯 Git 工作流，因此通过 `git-svn` 进行本地代码管理。面临的问题：
1. 没有所有代码权限，只能通过 git-svn 分别拉取各服务的代码仓库
2. 每次同步拉取最新代码都需要去各个服务目录操作
3. 每次提交也需要逐一进入各服务目录推送

示例项目结构：
- `service-config` 配置模块
- `service-web` Web 层代码
  - `service-app` 应用服务代码
  - `service-sysconf` 系统配置服务代码
  - ...
- `service-server` Server 层代码
- `service-common` 公共模块
- ...

### 以前的 clone 代码操作
```bash
git svn clone svn://svn.example.com/repos/project/trunk/service-config -r HEAD
git svn clone svn://svn.example.com/repos/project/trunk/service-web -r HEAD
```

### 以前更新代码操作
```bash
cd service-web && git svn rebase
```

### 以前提交代码操作
```bash
cd service-web && git svn dcommit
```

### 需求
我想做一个 VS Code 插件：
- 来简化 git-svn 操作，实现一键同步代码，一键提交代码，一键更新代码等操作；
- 支持切换分支
- 支持 git worktree，实现多分支并行开发
